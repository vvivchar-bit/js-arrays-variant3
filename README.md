# Практична робота 7: Масиви

## Дисципліна

Основи програмування мовою JavaScript

## Тема

Масиви — методи, ітерація, алгоритми обробки даних

## Варіант 3

Task Tracker + Трансформації масивів

## Опис проєкту

У межах практичної роботи розроблено вебзастосунок **Task Tracker** для роботи із завданнями.

Програма дозволяє:

- додавати нові завдання;
- змінювати статус завдання;
- видаляти завдання;
- фільтрувати завдання за пріоритетом;
- фільтрувати завдання за статусом;
- переглядати статистику завдань;
- визначати прострочені завдання;
- демонструвати роботу функцій трансформації масивів.

## Структура завдання

Кожне завдання має такі поля:

- назва;
- пріоритет: high, medium, low;
- статус: pending, in-progress, done;
- дата виконання.

## Реалізовані функції Task Tracker

### Додавання завдання

Користувач вводить назву завдання, обирає пріоритет, статус і дату. Після натискання кнопки завдання додається до списку.

### Зміна статусу

Для кожного завдання доступні кнопки зміни статусу:

- «У процесі»;
- «Виконано».

### Видалення завдання

Кнопка «Видалити» прибирає завдання зі списку.

### Фільтрація

Реалізовано фільтрацію:

- за пріоритетом;
- за статусом.

Також є кнопка скидання фільтрів.

### Статистика

Програма показує:

- загальну кількість завдань;
- кількість виконаних;
- кількість завдань у процесі;
- кількість завдань, які очікують;
- кількість прострочених завдань.

## Функції трансформації масивів

У проєкті реалізовано такі функції:

### flatten(arr)

Рекурсивно розгортає вкладений масив.

### groupBy(arr, key)

Групує масив об’єктів за заданим полем.

### unique(arr)

Повертає унікальні елементи масиву зі збереженням порядку.

### chunk(arr, size)

Розбиває масив на частини заданого розміру.

## Використані методи масивів

У роботі використано сучасні методи JavaScript:

- map;
- filter;
- reduce;
- find;
- slice;
- indexOf.

## Збереження даних

Для збереження завдань використовується `localStorage`, тому після оновлення сторінки створені завдання не зникають.
